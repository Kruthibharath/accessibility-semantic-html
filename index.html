Accessibility Features: Easy Wins

Handy tips on how to create accessible websites for everyone

Using semantic HTML improves accessibility in many ways.

  1. Better for mobile applications: Requires less memory than non-semantic HTML. (All those id attributes can really increase file size.) Improves responsiveness because the browser can interpret the code better.
  2. Greatly benefits screen reader users: To ensure a smooth and seamless experience for everyone, including those with physical and cognitive disabilities.
  3. SEO: Semantic HTML helps improve SEO by making the content of websites more identifiable to search engines. Browsers can interpret the code better because it understands how to identify semantic HTML elements and they will not read id attributesbp.
